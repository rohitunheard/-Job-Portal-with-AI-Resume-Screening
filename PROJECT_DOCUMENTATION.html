1 Project Overview

The Job Portal is a full-stack web application built on the MERN stack (MongoDB, Express, React, Node.js). It provides three distinct user roles — Job Seekers, Employers, and Admins — each with dedicated features, a unified login system, and AI-powered resume screening via Google Gemini.

👤 Job Seeker

Register, login with OTP, browse 24+ jobs, apply in one tap, manage profile with resume & photo upload.

🏢 Employer

Create employer account, post/edit/delete job openings, toggle active/paused status. Jobs appear live to seekers.

🛡 Admin

Secure admin panel to view all job applications, user profiles, resumes, and run AI resume screening.

🤖 AI Screening

Admin can screen all uploaded resumes against any job role using Google Gemini AI and rank candidates by score.

7 Features In Detail

🔐 Unified Login System (3-Role)

Single /login page with a role selector. User picks Job Seeker / Employer / Admin, then sees the appropriate form. Each role has its own JWT token with a role claim.

Select Role
→
Enter Credentials
→
Receive OTP Email
→
Verify OTP
→
JWT Issued → Redirect
  • Passwords hashed with bcryptjs (salt rounds: 10)
  • OTP generated as 6-digit random number, hashed before storage, expires in 10 minutes
  • JWT contains: id, name/username, email, role — signed with JWT_SECRET, expires in 7 days
  • Admin uses username/password only (no OTP) for faster access
📄 User Profile & Resume Upload

After login, users can upload a profile photo and PDF resume, add bio, LinkedIn URL, and qualifications. Files are stored in backend/uploads/ and served as static files.

  • Multer: handles multipart/form-data with disk storage strategy
  • Profile photos shown in Navbar icon and Profile page
  • Resume viewable as link from profile page and admin panel
  • Upsert strategy: one profile per email (create if not exists, update if exists)
💼 Job Listings (Static + Live)

Jobs page shows 24 pre-loaded static jobs AND live employer-posted jobs fetched from MongoDB. All salaries in INR. Employer-posted jobs are marked with an "Employer Posted" badge.

  • Static jobs: hardcoded in Jobs.jsx and Foundjobs.jsx arrays
  • Live jobs: fetched from GET /api/jobpostings on page load
  • Foundjobs page includes a real-time search bar filtering by title, company, or location
  • Apply Now button sends application to MongoDB via POST /api/jobapplications
🏢 Employer Dashboard

Employers get a full-featured dashboard to manage job postings after login.

  • Post new jobs with title, description, location, type, salary, skills
  • Edit existing postings inline
  • Pause / Activate toggle — paused jobs hidden from seekers
  • Delete postings with confirmation
  • Stats cards: Total Postings, Active, Paused
  • Posted jobs appear instantly on /jobs and /foundjobs for all logged-in seekers
🛡 Admin Panel

Admins access a protected dashboard with 3 tabs:

  • Job Applications tab: full table of all applications with name, email, job, company, location, date
  • User Profiles tab: card view of all profiles with photo, bio, qualifications, LinkedIn, resume download
  • AI Resume Screening tab: enter a job role, click Run — system fetches all uploaded resumes, sends each to Gemini AI, ranks candidates by score with progress bar
🤖 AI Resume Screening (Gemini)

The admin AI screening feature works as follows:

Admin enters job role
→
Fetch all PDFs from /uploads
→
pdf-parse extracts text
→
Gemini AI analyses
→
Ranked results shown
  • Model used: gemini-1.5-flash (fast, free tier available)
  • AI returns structured JSON: score (0-100), grade (A-F), summary, strengths[], weaknesses[], missingSkills[], suggestions[], verdict
  • Results sorted by score descending — top candidate highlighted with gold badge
  • Each result shows animated score ring, strengths, missing skills, and View Resume link

9 Authentication & Security

JWT Token Strategy
RoleStorageToken KeyRedirect After Login
Job SeekerlocalStoragejobPortalUser (user object)/jobs
EmployerlocalStorageemployerUser + employerToken/employer/dashboard
AdminsessionStorageadminUser/admin/panel
Security Measures
  • bcryptjs: all passwords hashed with 10 salt rounds before storage
  • OTP: 6-digit random code, SHA-256 hashed before DB storage, 10-minute expiry
  • JWT: signed with JWT_SECRET from .env, includes role claim for route protection
  • JWT Middleware: backend/middleware/auth.js — verifyToken, authUser, authEmployer, authAdmin
  • Environment variables: all secrets (DB URL, JWT secret, SMTP creds, Gemini API key) stored in .env — never in code
  • Auth guards: frontend pages check localStorage on mount, redirect to /login if no session

12 How to Run the Project

Prerequisites
  • Node.js v18+ installed
  • MongoDB running locally on port 27017 (or MongoDB Atlas connection string)
  • Gmail account with App Password enabled (for OTP emails)
  • Google Gemini API key from aistudio.google.com/app/apikey (free)
Step 1 — Start Backend
cd backend
npm install
# Edit .env with your MongoDB URL, SMTP credentials, and Gemini API key
npm start
# Server running on http://localhost:5000
Step 2 — Start Frontend
cd frontend
npm install
npm run dev
# App running on http://localhost:5173
